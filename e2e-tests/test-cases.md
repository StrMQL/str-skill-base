# Skill Base E2E 测试用例

## 测试环境配置

- **测试 URL**: `http://localhost:8000`
- **浏览器**: Chromium (Playwright)
- **测试框架**: Playwright Test

---

## 测试用例列表

### 1. 系统初始化流程

#### TC-001: 首次访问系统初始化
**优先级**: P0 (Critical)
**前置条件**: 系统未初始化（无管理员账号）
**测试步骤**:
1. 访问首页 `/`
2. 验证页面自动跳转到初始化页面 `/init`
3. 验证页面显示"系统初始化"标题
4. 验证显示用户名和密码输入框
5. 验证显示"创建管理员账号"按钮

**预期结果**:
- 页面正确跳转到初始化页面
- 所有表单元素正常显示

---

#### TC-002: 创建管理员账号成功
**优先级**: P0 (Critical)
**前置条件**: 系统未初始化
**测试步骤**:
1. 访问 `/init`
2. 输入用户名: `admin`
3. 输入密码: `admin123`
4. 点击"创建管理员账号"按钮
5. 等待页面跳转

**预期结果**:
- 显示"初始化成功"提示
- 自动跳转到登录页面 `/login`

---

#### TC-003: 创建管理员账号验证失败
**优先级**: P1 (High)
**前置条件**: 系统未初始化
**测试步骤**:
1. 访问 `/init`
2. 输入用户名: `ad` (少于3个字符)
3. 输入密码: `12345` (少于6个字符)
4. 点击"创建管理员账号"按钮

**预期结果**:
- 显示验证错误提示
- 用户名错误: "用户名长度必须在 3-50 个字符之间"
- 密码错误: "密码长度至少 6 位"

---

### 4. Skill 发布流程

#### TC-012: 访问发布页面
**优先级**: P1 (High)
**前置条件**: 用户已登录
**测试步骤**:
1. 登录系统
2. 点击"发布 Skill"按钮
3. 等待页面加载

**预期结果**:
- 跳转到发布页面 `/publish`
- 显示发布表单
- 显示文件上传区域

---

#### TC-013: 发布新 Skill
**优先级**: P0 (Critical)
**前置条件**: 用户已登录，准备好 ZIP 文件
**测试步骤**:
1. 访问 `/publish`
2. 输入 Skill ID: `test-skill`
3. 输入 Skill 名称: `Test Skill`
4. 输入描述: `This is a test skill`
5. 上传包含 SKILL.md 的 ZIP 文件
6. 输入版本变更日志: `Initial release`
7. 点击"发布"按钮

**预期结果**:
- 显示发布成功提示
- 显示"查看详情"和"继续发布"按钮
- 可以跳转到新发布的 Skill 详情页

---

#### TC-014: 发布 Skill 版本更新
**优先级**: P1 (High)
**前置条件**: 用户已登录，拥有 Skill 的发布权限
**测试步骤**:
1. 访问 `/publish`
2. 输入已存在的 Skill ID
3. 上传新版本的 ZIP 文件
4. 输入变更日志
5. 点击"发布"按钮

**预期结果**:
- 新版本发布成功
- Skill 详情页显示新版本

---

#### TC-015: 发布验证失败-缺少 SKILL.md
**优先级**: P2 (Medium)
**前置条件**: 用户已登录
**测试步骤**:
1. 访问 `/publish`
2. 上传不包含 SKILL.md 的 ZIP 文件

**预期结果**:
- 显示错误提示: "zip 中未找到 SKILL.md"
- 无法提交发布
