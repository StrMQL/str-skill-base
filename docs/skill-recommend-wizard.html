Skill 选型向导

通过几步选择，推断你最该先固化成 Skill的内容。覆盖技术岗与市场、销售、创始人、创作者等非技术主责；并参考 Agent-Skills4-什么值得做成Skill.md、 Claude-Skills-官方构建指南完整版.md，以及「创意发散 / 一人公司」等无法穷举但可落地的思路。

本页为离线单文件，可本地直接打开。正式写 Skill 时请遵守官方指南：文件夹 kebab-case、根文件必须名为 SKILL.md、技能目录内不要放 README（说明放进正文或 references/）。非技术 Skill 同样适用「窄 description + 触发测试」；创意类避免一个 Skill 包揽「所有写作」。
